Zapier and Make let you connect Join to the rest of your hiring stack without writing a single line of code. Use Join events — like a new application, a stage change, or an accepted offer — as triggers to fire actions in Slack, your HRIS, a spreadsheet, or any of thousands of other apps. Set up a workflow once and let it run in the background every time a matching event occurs.

<Note>
  Zapier and Make integrations require the Advanced or Enterprise plan.
</Note>

## Connect Zapier to Join

<Steps>
  <Step title="Open the Zapier integration settings">
    Go to **Settings → Integrations → Zapier**.
  </Step>
  <Step title="Copy your API key">
    Copy your Join API key from the integration settings page.
  </Step>
  <Step title="Find the Join app in Zapier">
    In Zapier, create a new Zap and search for the **Join** app.
  </Step>
  <Step title="Connect your account">
    Click **Connect Account** and paste your API key when prompted.
  </Step>
  <Step title="Choose a trigger and build your Zap">
    Select a trigger event from Join and add the actions you want to run in your connected apps.
  </Step>
</Steps>

## Connect Make to Join

<Steps>
  <Step title="Open the Make integration settings">
    Go to **Settings → Integrations → Make**.
  </Step>
  <Step title="Copy your API key">
    Copy your Join API key from the integration settings page.
  </Step>
  <Step title="Find the Join module in Make">
    In Make, add a new module to your scenario and search for **Join**.
  </Step>
  <Step title="Create a new connection">
    Create a new connection and paste your API key when prompted.
  </Step>
  <Step title="Build your scenario">
    Use Join as a trigger or action and connect it to the other modules in your scenario.
  </Step>
</Steps>

## Available trigger events

Join fires the following events that you can use as triggers in Zapier or Make.

| Event | When it fires |
|---|---|
| New application received | A candidate submits an application |
| Candidate stage changed | A candidate moves to a new pipeline stage |
| Offer accepted | A candidate accepts an offer |
| Offer declined | A candidate declines an offer |

## Common use cases

Once you've connected Join, here are some workflows your team can set up right away.

- **Slack notification on new application**: fire a Slack message to #hiring whenever a candidate applies to one of your open roles, so your team stays in the loop in real time.
- **HRIS record creation on hire**: when a candidate accepts an offer, automatically create a new employee record in your HRIS so onboarding can begin without manual data entry.
- **Team email on offer acceptance**: notify the hiring manager the moment their candidate says yes, without anyone needing to check Join.
- **Spreadsheet logging**: append every new application's details to a Google Sheet so you have a running report for stakeholders.

<Tip>
  For more advanced conditional logic — for example, routing notifications by department or seniority — Make's visual scenario builder offers more flexibility than Zapier.
</Tip>
