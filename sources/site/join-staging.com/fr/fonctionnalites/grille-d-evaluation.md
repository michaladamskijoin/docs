## Une grille par poste. Remplie dans l'application, avec des notes.

Définissez les critères que chaque participant·e doit noter : « maîtrise de JavaScript », « system design », « adéquation avec l'équipe », selon ce que le poste exige. Chacun·e remplit la grille directement dans Join, pendant ou après l'entretien, avec des commentaires libres pour chaque critère. La même grille pour tout le panel : pas de notation a posteriori, aucun retour qui manque.

- Définir les critères avant l'entretien, pas après
- Les participant·es remplissent la grille pendant ou juste après l'entretien
- Un commentaire libre par critère, à côté de la note

![Profil candidat·e dans Join affichant les notes agrégées de la grille d'évaluation du panel et les commentaires des participant·es](https://join-staging.com/_astro/scorecards-2-en.B7AjUEMA_Msbi1.webp)

02

## Les notes du panel s'affichent sur le profil candidat, pas dans un tableur.

À la clôture du tour, les notes de tou·tes les participant·es se regroupent sur le profil du·de la candidat·e, à côté de leurs commentaires. Les désaccords sont visibles et peuvent se traiter. Pour comparer les candidat·es, on navigue entre les profils : les données sont là, même si la vue comparative entre candidat·es n'existe pas encore.

- Les notes de chaque participant·e se regroupent sur le profil du·de la candidat·e
- Les désaccords sont visibles et se règlent au même endroit
- Pas de tableur, aucun retour à relancer une fois le tour terminé
