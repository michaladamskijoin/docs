The candidates resource gives you read access to the applicant data in your Join workspace. Use it to export candidates to your HRIS, build source-of-hire reports in your BI tool, or trigger downstream workflows when a candidate reaches a specific pipeline stage.

## List candidates

Returns a paginated list of all candidates across the jobs you have access to.

**Endpoint**

```
GET /v1/candidates
```

### Query parameters

<ParamField query="job_id" type="string">
  Filter results to candidates who applied to a specific job. Use the job's `id` value from the [Jobs resource](/api/jobs).
</ParamField>

<ParamField query="stage" type="string">
  Filter by pipeline stage slug (for example, `phone-screen` or `final-interview`). Stage slugs match the identifiers returned in the job's `pipeline_stages` array.
</ParamField>

<ParamField query="status" type="string">
  Filter by candidate status. Accepted values: `active`, `rejected`, `hired`, `archived`.
</ParamField>

<ParamField query="limit" type="integer">
  Number of results to return per page. Defaults to `50`; maximum is `200`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor returned in the previous response. Pass this value to retrieve the next page of results.
</ParamField>

### Sample request

```bash
curl "https://api.join.com/v1/candidates?job_id=job_abc123&status=active&limit=50" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Get a candidate

Returns the full profile for a single candidate.

**Endpoint**

```
GET /v1/candidates/{id}
```

### Path parameter

<ParamField path="id" type="string" required>
  The unique identifier of the candidate (for example, `cand_01hx9z2k3m`).
</ParamField>

### Sample request

```bash
curl https://api.join.com/v1/candidates/cand_01hx9z2k3m \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Sample response

```json
{
  "id": "cand_01hx9z2k3m",
  "first_name": "Maria",
  "last_name": "Schmidt",
  "email": "maria.schmidt@example.com",
  "phone": "+49 30 12345678",
  "location": "Berlin, Germany",
  "job_id": "job_abc123",
  "stage": "phone-screen",
  "status": "active",
  "applied_at": "2024-06-10T09:14:00Z",
  "source": "linkedin",
  "cv_url": "https://files.join.com/cvs/cand_01hx9z2k3m.pdf",
  "tags": ["senior", "remote-ok"]
}
```

## Candidate fields reference

| Field | Type | Description |
|---|---|---|
| `id` | string | Unique candidate identifier |
| `first_name` | string | Candidate's first name |
| `last_name` | string | Candidate's last name |
| `email` | string | Primary email address |
| `phone` | string | Phone number, if provided |
| `location` | string | Location as entered by the candidate |
| `job_id` | string | ID of the job the candidate applied to |
| `stage` | string | Current pipeline stage slug |
| `status` | string | `active`, `rejected`, `hired`, or `archived` |
| `applied_at` | string (ISO 8601) | Timestamp when the application was submitted |
| `source` | string | Originating job board or channel (e.g., `linkedin`, `join_job_board`, `referral`) |
| `cv_url` | string | Signed URL to the candidate's CV file |
| `tags` | array of strings | Tags applied to the candidate in Join |

<Note>
  The API returns only candidates for jobs your API key's account has access to, based on your roles and permissions settings.
</Note>

<Tip>
  Use the `source` field to identify which job board a candidate came from. This is useful for building source-of-hire reports in your BI tool — group by `source` to see which channels produce the most hires.
</Tip>
