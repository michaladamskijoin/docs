The Join REST API gives you programmatic access to your hiring data — job postings, candidate profiles, pipeline stages, and more. Use it to build custom integrations, push data to your HRIS, or pull hiring metrics into your business intelligence stack. The API is available on the **Advanced** and **Enterprise** plans.

## Base URL

All API requests go to the following base URL:

```
https://api.join.com/v1
```

## Authentication

Every request must include your API key as a Bearer token in the `Authorization` header. Generate your key in **Settings → Integrations → API**.

```bash
Authorization: Bearer YOUR_API_KEY
```

See the [Authentication guide](/api/authentication) for step-by-step instructions on generating your key, securing it, and handling auth errors.

## Common use cases

- Export candidate data to your HRIS automatically when an offer is accepted
- Pull hiring metrics into Looker, Metabase, Tableau, or Power BI
- Build a custom integration with your internal HR tools
- Power automation scenarios via Zapier or Make

## Quick start

Make your first request by replacing `YOUR_API_KEY` with the key from **Settings → Integrations → API**:

```bash
curl https://api.join.com/v1/jobs \
  -H "Authorization: Bearer YOUR_API_KEY"
```

A successful response returns a paginated JSON object containing your workspace's job postings.

## Rate limits

Join enforces standard REST API rate limits on all requests. The API returns a `429 Too Many Requests` status when you exceed the limit; check the `Retry-After` response header for how long to wait before retrying.

<Note>
  Contact [Join support](https://help.join.com) for details on specific rate limit thresholds or to discuss high-volume use cases.
</Note>

## Plan requirement

<Note>

</Note>
