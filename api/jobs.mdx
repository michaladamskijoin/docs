The jobs resource gives you read access to job posting data in your Join workspace. Use it to sync open roles to an internal dashboard, feed live vacancies into a custom careers page, or join job data with candidate records in your BI tool.

## List jobs

Returns a paginated list of jobs in your workspace.

**Endpoint**

```
GET /v1/jobs
```

### Query parameters

<ParamField query="status" type="string">
  Filter by job status. Accepted values: `online`, `offline`, `archived`. Omit this parameter to return jobs of all statuses.
</ParamField>

<ParamField query="limit" type="integer">
  Number of results to return per page. Defaults to `50`; maximum is `200`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor returned in the previous response. Pass this value to retrieve the next page of results.
</ParamField>

### Sample request

```bash
curl "https://api.join.com/v1/jobs?status=online&limit=50" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Get a job

Returns the full details for a single job posting.

**Endpoint**

```
GET /v1/jobs/{id}
```

### Path parameter

<ParamField path="id" type="string" required>
  The unique identifier of the job (for example, `job_abc123`).
</ParamField>

### Sample request

```bash
curl https://api.join.com/v1/jobs/job_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Job status reference

| Status | Meaning |
|---|---|
| `online` | Published and actively accepting applications |
| `offline` | Unpublished, but the hiring pipeline remains active and counts as an active job |
| `archived` | Closed; no longer counts toward your plan's active job limit |

<Note>
  `archived` jobs are closed and no longer counted as active. Use the `status` filter to exclude archived jobs when syncing open roles.
</Note>

<Tip>
  Use the jobs API to sync your current open roles to an internal dashboard or company intranet, keeping employees informed of open positions without manual updates.
</Tip>
