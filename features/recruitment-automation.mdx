Join's visual workflow editor lets you automate the repetitive, low-judgment tasks that slow down every hiring team — sending rejection emails, tagging applicants by source, moving candidates after a knockout screen — so you can focus on the conversations that actually require a human. You build every workflow in a no-code canvas using a simple "when X happens → do Y" model.

## How workflows work

Every workflow follows the same structure: one trigger fires when a specific event occurs, and one or more actions run immediately in response. You can add conditional branches to route candidates down different paths based on their attributes — score, location, department, or language — so a single workflow can handle multiple scenarios without duplicating logic.

- **One trigger** defines the event that starts the workflow.
- **One or more actions** run automatically when the trigger fires.
- **Conditional branches** (optional) let you route candidates differently based on their data.

## Triggers

| Trigger | When it fires |
|---|---|
| **Stage move** | A candidate moves into a specific pipeline stage |
| **Source board** | A candidate applied from a specific job board |
| **Application form answer** | A screening question answer matches a condition (e.g., knockout pass/fail) |
| **Manual** | A recruiter clicks the workflow button on a candidate profile |

## Actions

| Action | What it does |
|---|---|
| **Send email** | Sends a templated email with merge fields filled from the candidate record |
| **Change stage** | Moves the candidate to a different pipeline stage |
| **Add tag** | Applies a tag to the candidate for filtering and reporting |

## Build a workflow

<Steps>
  <Step title="Open Automations">
    Go to **Settings → Automations**.
  </Step>
  <Step title="Create a new workflow">
    Click **New workflow** to open the visual editor.
  </Step>
  <Step title="Select a trigger">
    Choose the event that should start the workflow — a stage move, source board, form answer, or manual button.
  </Step>
  <Step title="Add actions">
    Add one or more actions to run when the trigger fires. Drag to reorder them.
  </Step>
  <Step title="Add conditions (optional)">
    Click **Add branch** to route candidates differently based on score, location, department, or language.
  </Step>
  <Step title="Activate">
    Click **Activate** to switch the workflow on. It will fire on all matching events going forward.
  </Step>
</Steps>

## Example: rejection email on stage move

Here's what a simple automated rejection looks like in practice.

- **Trigger:** Candidate moves to the "Rejected" stage
- **Action:** Send the "Rejection — thank you" email template
- **Result:** Every rejected candidate receives a timely, personalised rejection with no recruiter action required

Add a second action to also apply a "Rejected" tag and you have a clean, searchable record of every rejection without any manual tagging.

<Tip>
  Chain multiple actions in one workflow: move the stage AND send an email AND add a tag — all from a single trigger. You don't need three separate workflows.
</Tip>

<Info>
  For Slack notifications, HRIS updates, or other third-party tools, use the Zapier or Make integration. Join events fire as triggers in your Zapier or Make scenarios, so you can connect them to any app in those ecosystems.
</Info>
